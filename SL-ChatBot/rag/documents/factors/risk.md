# 리스크 팩터 (Risk Factors)

## 개요
리스크 팩터는 주식의 변동성과 위험을 측정하여 포트폴리오 관리에 활용합니다.

## 주요 팩터

### 1. 변동성 (Volatility)
- **정의**: 주가 수익률의 표준편차
- **공식**: `σ = √(Σ(수익률 - 평균)² / n)`
- **단위**: 연율화 (%)
- **해석**: 높을수록 위험
- **기준값**:
  - 낮음: 15% 이하
  - 보통: 15-30%
  - 높음: 30% 이상

### 2. 베타 (Beta)
- **정의**: 시장 대비 민감도
- **공식**: `β = Cov(주식, 시장) / Var(시장)`
- **해석**:
  - β = 1: 시장과 동일
  - β > 1: 시장보다 변동 큼 (공격적)
  - β < 1: 시장보다 변동 작음 (방어적)
  - β < 0: 시장과 반대 (헤지)

### 3. MDD (Maximum Drawdown)
- **정의**: 최고점 대비 최대 하락률
- **공식**: `(저점 - 고점) / 고점 × 100`
- **관리 목표**:
  - 보수적: -15% 이내
  - 중립: -20% 이내
  - 공격적: -30% 이내

### 4. VaR (Value at Risk)
- **정의**: 일정 신뢰수준의 최대 손실 예상액
- **예시**: 95% 신뢰수준에서 1일 VaR 2%
- **의미**: 5% 확률로 2% 이상 손실

### 5. 샤프 비율 (Sharpe Ratio)
- **정의**: 위험 대비 초과 수익
- **공식**: `(수익률 - 무위험수익률) / 변동성`
- **기준값**:
  - 우수: 1.5 이상
  - 양호: 1.0-1.5
  - 보통: 0.5-1.0

## 리스크 관리 기법

### 1. 포지션 사이징
```python
# 변동성 기반 포지션 크기
position_size = target_risk / volatility

# 예: 포트폴리오의 2%를 목표 리스크로
# 변동성 30% 종목: 2% / 30% = 6.7% 비중
# 변동성 15% 종목: 2% / 15% = 13.3% 비중
```

### 2. 손절매 (Stop Loss)
```
- 고정 손절: -10%
- 변동성 기반: -2σ
- 이동평균 기반: 20일 MA 하회
```

### 3. 분산투자
```
- 종목 수: 20-30개
- 섹터 분산: 5개 이상
- 개별 종목: 최대 10%
- 섹터별: 최대 30%
```

### 4. 헤징
```
- 인버스 ETF
- 옵션 (풋옵션 매수)
- 베타 0 이하 종목
```

## 시장 환경별 리스크 관리

### 저변동성 장세
- **전략**: 공격적 투자
- **베타**: 1.2 이상
- **레버리지**: 고려 가능

### 고변동성 장세
- **전략**: 방어적 투자
- **베타**: 0.8 이하
- **현금 비중**: 30% 이상

### VIX 지수 활용
```
VIX < 15: 안정 (공격적)
VIX 15-20: 보통 (중립)
VIX 20-30: 불안 (방어적)
VIX > 30: 공포 (초방어적 또는 역발상)
```

## 저변동성 이상 현상

### Low Volatility Anomaly
- **현상**: 저변동성 주식이 장기적으로 초과수익
- **이유**:
  - 투자자의 복권 선호
  - 레버리지 제약
  - 벤치마크 추종

## 위험 조정 수익률 지표

### 1. 샤프 비율
```
Sharpe = (Rp - Rf) / σp
```

### 2. 소르티노 비율
```
Sortino = (Rp - Rf) / 하방변동성
```
- 하방 리스크만 고려

### 3. 칼마 비율
```
Calmar = 연평균수익률 / MDD
```

### 4. 정보 비율
```
Information Ratio = 초과수익 / 추적오차
```

## 리스크 팩터 활용 전략

### 1. 리스크 패리티
- 모든 자산의 리스크 기여도 동일
- 변동성 역가중

### 2. 최소 분산 포트폴리오
- 포트폴리오 변동성 최소화
- 상관관계 활용

### 3. 조건부 리스크 관리
```python
if MDD > 15%:
    reduce_position_by_50%
if 손실 > 10%:
    stop_loss()
```

## 주의사항

1. **과거 변동성 ≠ 미래 변동성**
   - 변동성은 시간에 따라 변화
   - 레짐 체인지 고려

2. **극단적 사건 (Fat Tail)**
   - 정규분포 가정의 한계
   - 블랙스완 이벤트

3. **상관관계 변화**
   - 위기 시 상관관계 급증
   - 분산 효과 감소

4. **거래 비용**
   - 리밸런싱 비용
   - 유동성 리스크
